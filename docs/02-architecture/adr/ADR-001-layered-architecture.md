# ADR-001: Многослойная архитектура

## Статус

Принято

## Контекст

Курсовой проект требует явного разделения ответственности. Приложение Fly
сочетает UI, бизнес-логику и доступ к данным.

## Решение

Применить многослойную архитектуру: Presentation → Control → Model → Entity/Foundation.
Соответствует рекомендациям Flutter и методическим материалам по программной инженерии.

## Последствия

- (+) Понятная структура каталогов, простота сопровождения
- (+) Возможность подмены `SqliteTaskRepository` на mock для тестов
- (−) Больше файлов по сравнению с монолитным подходом
