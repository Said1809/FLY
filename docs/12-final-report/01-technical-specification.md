# Техническое задание

Документ составлен в соответствии со структурой ГОСТ 19.201-78 «Техническое задание.
Требования к содержанию и оформлению».

## 1. Введение

### 1.1. Наименование программы

«Fly» — кроссплатформенное приложение для персонального и учебного планирования задач.

### 1.2. Краткая характеристика области применения

Программа предназначена для студентов и пользователей, ведущих учебную и личную
деятельность. Область применения — образование, самоорганизация, тайм-менеджмент.

## 2. Основание для разработки

Разработка ведётся в рамках курсового проекта по дисциплине «Программная инженерия»
(СКФУ, направление 09.03.04), траектория кроссплатформенной разработки.

## 3. Назначение разработки

### 3.1. Функциональное назначение

Предоставление пользователю инструмента для создания задач с датой и временем,
навигации по недельному и месячному календарю, отметки выполнения и персонализации
интерфейса. Аутентификация — через Supabase Auth; данные задач — локально в SQLite.

### 3.2. Эксплуатационное назначение

Приложение эксплуатируется конечными пользователями на мобильных устройствах (Android),
ПК (Windows) и в браузере (Web).

## 4. Требования к программе

### 4.1. Требования к функциональным характеристикам

1. Регистрация и аутентификация пользователей (email/password, Supabase).
2. CRUD операции над задачами (создание, чтение, обновление, удаление).
3. Привязка задачи к календарной дате и опциональному времени.
4. Недельное представление с навигацией свайпом.
5. Месячный календарь с индикацией статуса дней.
6. Отметка задачи выполненной / снятие отметки.
7. Настройка темы (светлая/тёмная), шрифта и акцентного цвета.
8. Русская локализация интерфейса и дат.

### 4.2. Требования к надёжности

- Локальные данные сохраняются в SQLite при каждой операции CRUD.
- Ошибки аутентификации отображаются пользователю.
- При отсутствии сети задачи доступны для просмотра и редактирования.

### 4.3. Требования к составу и параметрам технических средств

| Компонент | Требование |
|-----------|------------|
| Клиент Android | Android 8.0+, 100 МБ свободного места |
| Клиент Web | Chrome / Edge последних версий |
| Клиент Windows | Windows 10+, 200 МБ RAM |
| Облако | Аккаунт Supabase (Auth) |

### 4.4. Требования к информационной и программной совместимости

- Клиент: Dart 3, Flutter 3.x, Material Design 3.
- Локальная БД: SQLite через sqflite.
- Аутентификация: Supabase Flutter SDK.

### 4.5. Требования к безопасности

- Пароли не хранятся локально в открытом виде.
- Сессия Supabase управляется SDK.
- Задачи хранятся только на устройстве пользователя.
