# Спецификации прецедентов

## UC2 — Войти в систему

| Поле | Значение |
|------|----------|
| **ID** | UC2 |
| **Название** | Вход в систему |
| **Актор** | Пользователь |
| **Предусловия** | Приложение установлено; есть учётная запись Supabase |
| **Постусловия** | Сессия активна; отображается `HomePage` |

### Основной сценарий

1. Пользователь открывает приложение.
2. `AuthGate` проверяет `currentSession` — сессии нет.
3. Отображается `AuthScreen`.
4. Пользователь вводит email и пароль.
5. Система вызывает `Supabase.auth.signInWithPassword`.
6. Supabase возвращает сессию.
7. `onAuthStateChange` обновляет состояние `_isLoggedIn = true`.
8. `AuthGate` отображает `HomePage`.

### Альтернативные сценарии

| Шаг | Условие | Действие |
|-----|---------|----------|
| 5a | Неверные учётные данные | Показать SnackBar с текстом ошибки |
| 5b | Нет сети | Показать сообщение о недоступности сервиса |

## UC3 — Создать задачу

| Поле | Значение |
|------|----------|
| **ID** | UC3 |
| **Название** | Создание задачи |
| **Актор** | Пользователь |
| **Предусловия** | Пользователь авторизован |
| **Постусловия** | Задача сохранена в SQLite; список на дату обновлён |

### Основной сценарий

1. На `HomePage` пользователь нажимает FAB «+».
2. Открывается `AddTaskPage`.
3. Пользователь вводит название (обязательно), описание, выбирает дату и опционально время.
4. Нажимает «Сохранить».
5. Система создаёт `Task` с `id = millisecondsSinceEpoch`.
6. `TaskController.addTask()` → `SqliteTaskRepository.insert()`.
7. Экран закрывается; `HomePage` перезагружает задачи.

### Альтернативные сценарии

| Шаг | Условие | Действие |
|-----|---------|----------|
| 4a | Пустое название | Кнопка «Сохранить» неактивна / показать ошибку валидации |

## UC5 — Отметить задачу выполненной

| Поле | Значение |
|------|----------|
| **ID** | UC5 |
| **Название** | Отметка выполнения |
| **Актор** | Пользователь |
| **Предусловия** | Задача отображается на текущей дате |
| **Постусловия** | Флаг `isCompleted` изменён в БД |

### Основной сценарий

1. Пользователь нажимает checkbox на `TaskCard`.
2. `TaskController.toggleComplete(task)` создаёт копию с инвертированным флагом.
3. `SqliteTaskRepository.updateTask()` сохраняет изменение.
4. UI обновляется: задача перемещается в секцию «Завершённые задачи».

### Альтернативный сценарий

| Шаг | Условие | Действие |
|-----|---------|----------|
| 1a | Задача уже выполнена | Снятие отметки (возврат в активные) |
