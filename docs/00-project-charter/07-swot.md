# SWOT-анализ

| | **Положительные** | **Отрицательные** |
|---|-------------------|-------------------|
| **Внутренние** | **Strengths (Сильные стороны)** | **Weaknesses (Слабые стороны)** |
| | — Кроссплатформенность (один код — Android, Web, Windows) | — Нет синхронизации задач между устройствами |
| | — Офлайн-доступ к задачам (SQLite) | — Задачи не привязаны к user_id в БД |
| | — Чёткая слоёная архитектура | — Тестовое покрытие не реализовано |
| | — Персонализация UI (тема, шрифт, цвет) | — AuthService не используется UI напрямую |
| | — Готовый BaaS для аутентификации (Supabase) | |
| **Внешние** | **Opportunities (Возможности)** | **Threats (Угрозы)** |
| | — Расширение: облачная синхронизация через Supabase DB | — Конкуренция с Todoist, Google Calendar |
| | — Push-уведомления о дедлайнах | — Зависимость от доступности Supabase |
| | — Публикация в Google Play / RuStore | — Изменения API Supabase / Flutter |
| | — Интеграция с учебным расписанием СКФУ | — Потеря локальных данных при смене устройства |

## Выводы

Проект опирается на зрелый стек Flutter и Supabase, что сокращает время разработки.
Основной риск — локальное хранение без привязки к пользователю; для курсового MVP
это допустимо и документируется как ограничение.
