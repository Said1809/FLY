# Руководство по настройке Supabase

## 1. Назначение

Supabase обеспечивает регистрацию и вход пользователей приложения Fly.
Сервер приложения разрабатывать не требуется.

## 2. Создание проекта

1. Зарегистрируйтесь на [supabase.com](https://supabase.com).
2. Создайте новый проект (выберите регион, задайте пароль БД).
3. Дождитесь инициализации проекта.

## 3. Получение ключей

В панели проекта: **Settings → API**

| Параметр | Куда вставить |
|----------|---------------|
| Project URL | `Supabase.initialize(url: ...)` в `lib/main.dart` |
| anon public key | `Supabase.initialize(anonKey: ...)` |

## 4. Настройка Authentication

**Authentication → Providers → Email**

- Включите Email provider.
- Для разработки можно отключить «Confirm email».
- Для production включите подтверждение email.

## 5. Проверка

```bash
flutter run
```

1. Зарегистрируйте тестового пользователя.
2. В Supabase Dashboard → **Authentication → Users** должен появиться пользователь.

## 6. Безопасность

- **Не коммитьте** service_role key в репозиторий — только anon key в клиенте.
- Row Level Security для таблиц Supabase в MVP не используется (задачи локальные).

## 7. Лимиты бесплатного тарифа

- Достаточно для курсового проекта и демонстрации.
- При превышении лимитов auth-запросов — upgrade или новый проект.
